# WeMD 调研结论

## 可直接吸收
1. 微信复制前 DOM 归一化：把最外层 padding、背景色和文本颜色转移到微信更容易保留的位置。
2. 表格复制兼容：宽表格使用横向滚动容器，单元格使用更紧凑字号和 padding。
3. 图片滑动组语法：用一行 Markdown 图片组生成横向滑动图片区域。
4. 本地历史记录：本项目当前是静态站点，第一阶段先把现有 localStorage 历史扩到 30 条并增加自动快照，后续再考虑 IndexedDB。

## 需要改造后吸收
1. Markdown 插件体系：WeMD 使用 markdown-it，本项目 md 页面目前使用 marked + 自定义富文本处理，不能直接复制，需要轻量实现或迁移解析器。
2. 主题处理器：WeMD 使用 CSS + juice 内联。本项目是单 HTML 静态页，暂不适合引入完整构建链。
3. 代码块 Mac 顶栏：本项目 md 页面已有类似实现，note 页面可按需同步。

## 暂不建议第一轮吸收
1. 多图床上传：官方图床、七牛、阿里云、腾讯云、S3 都涉及密钥和服务端/跨域问题。
2. Electron 桌面端能力：与当前 GitHub Pages 静态部署目标不一致。
3. 完整深色模式算法：价值高，但改动面较大，适合单独立项。
